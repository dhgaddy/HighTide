## wd_in LEF bug + full recovery (2026-05-15)

The cnn-sky130hd GRT-0116 was **not** a floorplan problem — it was the
bsg_fakeram `generate_lef.py` `wd_in` pin-direction bug (upper half of every
write-data bus emitted as `DIRECTION OUTPUT`; see CLAUDE.md workaround table).
This invalidated **every** pre-fix `_final` pass (Section A + B).

Recovery (done, pushed to `origin/fakeram`):
- Fork fix `c83ecb4` on `asap7-area-calib-v2`; HighTide submodule bumped `044c02b9`.
- `e2abcff0` — Section A regenerated (bp_processor, cnn, liteeth, litepci, NVDLA, ternip × all plats).
- `da068a64` — Section B coralnpu + NyuziProcessor; added `tools/reconstruct_cfg_from_lef.py` (their cfgs were missing/drifted).
- `77bf0ed4` — vortex: 6 unsizable macros → FF-fallback in `VX_dp_ram_REPLACE.sv`; 6 sizable regenerated.
- All regenerated LEFs verified: zero `DIRECTION OUTPUT` on any `*_wd_in` pin.

## How to drive a new build

```bash
# Single design × platform on k8s (branch must be pushed to origin first)
./k8s/run.sh --branch fakeram <platform> <design>

# All NVDLA partitions on one platform
for p in partition_a partition_c partition_m partition_o partition_p; do
  ./k8s/run.sh --branch fakeram asap7 $p
done

# Status
./k8s/run.sh --status

# Local rebuild (no k8s)
bazel build //designs/<platform>/<design>:<design>_final
```
